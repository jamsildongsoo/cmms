# CMMS 모바일 어플리케이션 요구사항 및 UI 설계서

본 문서는 리파워에너지 CMMS의 모바일 어플리케이션(Flutter 기반) 개발을 위한 요구사항과 주요 화면 설계를 정의합니다.
현장 작업자와 관리자의 편의성, 그리고 신속한 업무 처리를 최우선으로 고려하며 핵심 기능인 **결재함, QR 기반 예방점검 실적 등록, TM 메모 관리, 설비 정보 조회**에 집중합니다.

## 1. 개요 및 기술 환경

*   **Platform**: Android / iOS (Cross-platform)
*   **Framework**: Flutter (최신 안정화 버전)
*   **State Management**: 프로젝트 표준에 맞춰 Provider, Riverpod, 또는 GetX 도입 권장
*   **Network**: REST API 연동, JWT 기반 Bearer Token 인증 (Axios/Dio 패키지 활용)
*   **Hardware Interface**: 디바이스 카메라 (QR 코드 스캐너, 현장 사진 촬영)
*   **Target Device**: 스마트폰 (세로 모드(Portrait) 전용 UI 최적화)

## 2. 주요 기능 상세 요구사항

모바일 앱의 사용성을 고려하여 불필요한 입력은 최소화하고 직관적인 조작을 제공합니다.

### 2.1 결재함 (Approval Box)
*   **목적**: 이동 중이거나 외근/현장 업무 중에도 신속한 의사결정 및 문서 처리.
*   **기능 요건**:
    *   **결재 대상 문서 조회**: 결재선에 자신이 포함된 미결함 문서 목록을 제공.
    *   **문서 상세 조회**: 작업허가(PTW), 작업지시(WO), 예방점검 등 연관 문서 내용과 상세 정보를 모바일에 최적화하여 표시.
    *   **결재 처리**: 승인(결재/합의/통보) 및 반려 기능. 승인/반려 시 코멘트 필수 입력 또는 선택 입력 지원.
*   **연관 테이블**: `approval`, `approval_step`

### 2.2 예방점검 실적 등록 (QR 기반)
*   **목적**: 대상 설비 앞에서 QR 코드를 스캔하여 점검 대상을 즉시 확정하고 실적을 등록.
*   **기능 요건**:
    *   **QR 스캔 조회**: 설비에 부착된 QR 코드를 앱 내장 카메라로 스캔.
    *   **점검 목록 연동**: 
        *   점검 계획(Stage: PLN) 데이터를 기반으로 현시점에 작성해야 할 점검 대상 목록 매칭.
        *   해당 설비에 대한 정기 점검이 없을 경우, [불시 점검/긴급 실적 등록(Stage: ACT)] 신규 생성 기능 제공.
    *   **실측값 입력**: `inspection_item`에 명시된 항목별로 수치 입력(기준값 비교) 및 양호/불량 판정 폼 제공.
    *   **임시저장 기능 (필수)**: 
        *   상태값을 임시(`status='T'`)로 두고 저장하여, 점검 중 이동이나 작업 중단 시 내용이 날아가는 것을 방지.
        *   네트워크의 음영 지역을 대비하여 단말기 로컬DB에 자동 캐싱하는 방안도 고려 가능.
    *   **사진 첨부**: 카메라 직접 촬영 또는 갤러리 참조를 통해 설비 상태나 문제점 사진 즉시 업로드.
*   **연관 테이블**: `equipment`, `inspection`, `inspection_item`

### 2.3 TM 메모 (공지/현황 게시판)
*   **목적**: 당일 이슈, 교대 조달 사항, 긴급 전파 내용을 언제 어디서나 확인 및 소통.
*   **기능 요건**:
    *   **목록 무한 스크롤**: 최신 등록순으로 메모 목록 조회 제공 (페이징 방식 또는 무한스크롤).
    *   **상세 확인 및 댓글 기능**: 공지 내용 확인 후 조치 진행 상항이나 추가 현황을 1단계 댓글(`memo_comment`)로 기록.
    *   **작성 기능**: 모바일 환경에서도 간단한 텍스트와 사진 첨부 기반의 새로운 TM 메모 작성 지원.
*   **연관 테이블**: `memo`, `memo_comment`

### 2.4 설비 목록 및 상세 조회
*   **목적**: 현장 근무자가 특정 설비의 기본 제원 및 이력을 QR 코드로 한눈에 파악.
*   **기능 요건**:
    *   **조회 방식**: 
        1. 단순 목록 검색 (설비명/코드).
        2. UI 통합형 **QR아이콘 버튼**을 통해 카메라 뷰를 띄워 스캔.
    *   **상세 제원 표시**: 모바일 화면에 맞춰 제조사, 폼팩터, 스펙, 위치 등의 정보를 아코디언 메뉴나 카드형으로 표시.
    *   **보전 이력 표시**: 해당 설비와 연관된 최근의 예방 점검(`inspection`), 작업 지시(`work_order`) 이력을 타임라인 형태로 3~5건 요약 노출.
*   **연관 테이블**: `equipment`, `inspection`, `work_order`

## 3. UI/UX 화면 흐름 설계 (Screen Flow)

### 3.1 공통 네비게이션 체계
*   **Bottom Navigation Bar (하단 탭)**
    *   `[ 홈/대시보드 ]` : 나의 미결재 건수, 오늘 해야 할 예방점검 리스트, 주요 TM메모 최근 글 요약 제공.
    *   `[ 설비조회 ]` : 설비 목록 검색 화면. 검색창 우측에 크게 자리잡은 [QR 스캔] 아이콘 배치.
    *   `[ QR 스캔 (Main Action) ]` : 정중앙에 플로팅 액션 버튼(FAB) 또는 강조된 탭 구성으로 즉각적인 카메라 뷰 실행 지원. (예방점검 혹은 설비정보 조회 분기 처리)
    *   `[ 결재함 ]` : 미결함 / 기결함 탭 뷰 리스트 화면.
    *   `[ TM메모 ]` : 최신 활동 현황을 카드뷰 형태로 보여주는 화면.

### 3.2 핵심 화면 상호작용
*   **점검 실재(임시저장) 폼**: 
    1. QR 인식 성공 $\rightarrow$ 설비 정보와 점검 내역 모달 팝업.
    2. 점검 상세 입력 플로우 진입 시, 입력 데이터가 길 경우를 대비하여 상단에 스텝 바(Step progress) 또는 그룹화된 리스트 제공.
    3. 하단에는 플로팅 바 형태로 `[ 취소 ] / [ 사진 추가 ] / [ 임시저장 ] / [ 결재 상신 ]` 버튼 세트 제공. 결재 상신은 필수값이 모두 채워졌을 때만 활성화 시킴.

*   **결재 상세 폼**: 
    *   가로 스크롤 테이블 요소 등은 모바일에서 깨지지 않도록 Label-Value 형식으로 반응형 재배치 구현. (참조: `CMMS_UI.md`의 고밀도 분할 디자인을 1 Col Stacked 형식으로 적용).
    *   하단 영역에 `[ 반려 ]` (Red Button) / `[ 승인 ]` (Blue Button)을 상시 고정(Fixed Bottom)하여 접근성을 높임.

## 4. 모바일 특화 보안 및 확장 정책
1.  **네트워크 단절 대응(Offline-Support)**:
    *   지하 펌프실, 외곽 지역 등 네트워크 음영 지역에서 예방점검 입력 도중 튕김 현상을 최소화하도록 임시저장 데이터의 내부 로컬 캐싱(Shared_Preferences 혹은 SQLite)을 적극 반영.
2.  **권한 관리(Permission Request)**:
    *   앱 최초 실행 및 카메라 접근 시 명시적인 `Camera`, `Storage` 권한 승인 플로우 포함 필수.
3.  **세션 및 인증(Token Refresh)**:
    *   이동 중 IP가 변경되거나 장기 미사용 시 무효화된 토큰에 대해 인터셉터를 두어 Refresh Token으로 매끄럽게 재인증 하거나, 로그인 화면으로 우회하도록 강제 설계.
