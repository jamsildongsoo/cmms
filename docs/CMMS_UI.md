# **CMMS(유지보수 관리 시스템) UI 설계서**

본 문서는 리파워에너지 CMMS의 화면 구성, 메뉴 구조 및 주요 프로세스(설비, 예방점검, 결재 등)의 UI 설계를 정의합니다.

## **1\. 공통 레이아웃 구성**

### **1.1 로그인 화면**

* **중앙 정렬**: 회사 로고, ID(사번), PW 입력 필드.  
* **기능**: 로그인 상태 유지, 비밀번호 초기화 요청.

### **1.2 로그인 이후 메인 레이아웃**

* **헤더(Header)**:  
  * **플랜트 선택**: 사용자가 권한을 가진 플랜트(사업장) 전환 드롭다운.  
  * **알림 센터**: 결재 요청, 점검 알람(D-Day) 아이콘.  
  * **사용자 정보**: 소속 부서, 성명, 로그아웃 버튼.  
* **좌측 메뉴(Sidebar)**: 아코디언 형태의 대메뉴/소메뉴 구성.  
* **메인 영역(Main Area)**: 상단 Breadcrumb(홈 \> 대메뉴 \> 소메뉴), 본문 영역.

## **3\. 주요 화면 구성 예시**

### **3.1 \[설비 등록\] 화면 (EQUIP\_REG)**

설비의 기본 정보와 관리 기준을 설정하는 화면입니다.

* **상단 액션바**: \[목록\], \[저장\], \[초기화\]  
* **기본 정보**:  
  * 설비코드(자동생성), 설비명, 설비분류, 설치위치(L1\~L4).  
  * 제조사, 모델명, 시리얼번호, 설치일자.  
* **관리 정보**:  
  * 관리부서(담당), 점검주기(Daily/Weekly/Monthly/Quarterly/Yearly).  
  * 중요도(A/B/C), 상태(가동/정지/폐기).  
* **첨부파일**: 도면(PDF/DWG), 사진, 매뉴얼 업로드.

---

### **3.2 \[예방점검 계획 및 실적 등록\] 화면 (PM\_PLAN\_ACT)**

계획 수립과 현장 실적 입력을 통합적으로 처리합니다.

#### **A. 목록 화면 (List View)**

* **그리드 구성**: 점검명, 설비명, 예정일자, 상태(계획/완료), 담당자.  
* **특이 기능**: 상태가 '계획(Plan)'인 행에는 **\[실적입력\]** 버튼이 활성화되어, 클릭 시 해당 계획 데이터가 포함된 실적 등록 화면으로 바로 진입.

#### **B. 등록 화면 (Entry Form)**

* **구분 선택 (Radio)**: \[ ● 계획 등록 \] / \[ ○ 실적 직접입력 \]  
* **계획 등록 모드 시**:  
  * 점검 주기, 시작 예정일, 반복 여부, 점검 항목(Standard Item) 설정.  
* **실적 직접입력 모드 시** (계획 없이 긴급 점검 등을 수행할 경우):  
  * 실제 점검 일시, 실제 작업자, 점검 결과 수치(Value), 판정(Pass/Fail) 입력.  
  * **계획 연동 시**: 기존 계획의 점검 항목이 자동으로 리스트업되어 결과값만 입력하면 됨.  
* **결과 입력 섹션**:  
  * 점검 항목별 체크리스트(OK/NG).  
  * 특이사항 기록 및 현장 사진 첨부.

---

### **3.3 \[결재관리\] 화면 (APPROVAL)**

업무 프로세스의 투명성을 위한 결재 UI입니다.

* **결재함 구성**:  
  * **미결함**: 내가 결재해야 할 문서 (Badge로 건수 표시).  
  * **기결함**: 내가 결재 완료한 문서.  
  * **반려함**: 내가 상신했으나 반려된 문서.  
  * **통보함**: 결재 프로세스에서 참조(통보)된 문서.  
* **결재 작성**:  
  * 결재선 지정(결재/합의/통보).  
  * 참조 문서(설비, 작업오더, 예방점검 실적 등) 링크 연동.

---

## **4\. UI/UX 디자인 원칙**

1. **일관성**: 모든 등록 화면은 상단에 액션 버튼을 배치하고 좌측에 기본 정보, 우측에 부가 정보를 배치함.  
2. **가독성**: 목록 화면에서 '긴급', '불합격(NG)', '미결' 등 주요 상태는 상태별 컬러(Red/Yellow/Blue)를 적용하여 시인성 확보.  
3. **편의성**: 실적 등록 시 모바일 기기에서의 입력을 고려하여 버튼 크기를 충분히 확보하고 사진 촬영 업로드 기능을 전면에 배치.
